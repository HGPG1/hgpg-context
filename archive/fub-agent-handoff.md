# HANDOFF DOCUMENT: FUB AI AGENT & TEXTING INTEGRATION

> Generated by Manus before decommissioning the original FUB AI Agent project (May 2026).
> Saved here as the source-of-truth spec for the planned Next.js + Supabase rebuild at `leads.homegrownpropertygroup.com`.

## 1. PURPOSE
The **FUB AI Agent** and **Texting Integration** were built as a unified suite to automate lead management and SMS communication within the Follow Up Boss (FUB) CRM for Home Grown Property Group.

*   **AI Agent**: Automates lead scoring (0-100), urgency classification (A/B/C), and pipeline cleanup (Active Client safeguards).
*   **Texting Integration**: A self-hosted "Texting Betty" replacement that sends scheduled SMS based on tags in FUB notes (e.g., `[NextDay]`, `[11:30AM]`).

## 2. ARCHITECTURE
The system consists of two primary services, often deployed together.

| Component | Role | Stack |
| :--- | :--- | :--- |
| **FUB AI Agent** | Backend engine for scoring and stage management. | Python, FastAPI, SQLAlchemy |
| **Texting Integration** | Full-stack SMS automation platform with admin dashboard. | React 19, Express, tRPC, Drizzle |
| **Database** | Stores scores, decisions, message history, and schedules. | MySQL/TiDB (Texting) / SQLite (Agent) |
| **Message Queue** | Handles delayed SMS delivery and retries. | Redis + Bull |
| **External APIs** | CRM and Communication providers. | Follow Up Boss API, Twilio API |

## 4. FUB INTEGRATION
The system uses a "Pull" (Agent) and "Push" (Texting) hybrid model.

*   **Auth**: HTTP Basic Auth (API Key as username).
*   **Webhooks**: Texting Integration listens for `notesCreated` events at `/api/webhooks/fub/:accountId`.
*   **Endpoints Used**:
    *   `GET /people`: Bulk lead fetching for scoring.
    *   `PUT /people/{id}`: Updating stages and classification tags.
    *   `POST /textMessages`: Syncing sent SMS back to FUB natively.
    *   `POST /notes`: Logging AI reasoning and SMS activity.
    *   `GET /events`: Analyzing property views/saves for scoring.

## 5. TAGGING & CLASSIFICATION LOGIC
### AI Scoring & Classification
*   **Scoring (0-100)**: Appointment (+30), Reply (+20), Property View (+10), No activity 30d (-15).
*   **A - Hot**: Appointment booked OR 5+ property views in 14 days OR Score >= 70.
*   **B - Warm**: Score >= 40 with activity in last 30 days.
*   **C - Cold**: No activity for 45+ days.

### SMS Timing Tags (Texting Integration)
Added to FUB Action Plan notes to trigger automation:
*   `[Scheduled]`: Send immediately (if within business hours).
*   `[11:30AM]`: Send at specific time today/tomorrow.
*   `[NextDay]`: Send next business day at opening time.
*   `[+2days]`: Send after 48-hour delay.
*   `[+30minutes]`: Send after 30-minute delay.

### SMS Merge Fields
*   `{{firstName}}`, `{{lastName}}`, `{{city}}`, `{{state}}`, `{{zip}}`.

## 6. WHAT WORKED
*   **Deterministic Logic**: Using a scoring engine instead of LLMs for every lead saved massive API costs and improved speed.
*   **Business Hours**: Prevents sending automated texts at 3 AM, preserving the "human" feel.
*   **Merge Fields**: High personalization led to better reply rates.
*   **Note-Based Triggers**: Allowed agents to control automation from within the FUB UI they already use.

## 7. WHAT DIDN'T WORK
*   **Rate Limits**: The initial scan of 12,000+ leads hit FUB limits frequently. Rebuild should use a distributed queue.
*   **Offset Pagination**: FUB's API gets slow/unreliable with high offsets. Always use `next_token` (keyset) pagination.
*   **Redis Dependency**: If Redis went down, scheduled messages were lost. Rebuild should use a persistent DB-backed queue (like Supabase pg_cron or Graphile Worker).

## 10. LESSONS LEARNED (FOR NEXT.JS + SUPABASE)
1.  **Supabase Realtime/Webhooks**: Use these to trigger scoring the moment a lead is updated, rather than polling.
2.  **Edge Functions**: Move the Timing Tag Parser to an Edge Function for low-latency SMS scheduling.
3.  **Database Queue**: Use Supabase's built-in scheduling or a table-based queue to avoid the overhead of Redis.
4.  **Native FUB Custom Fields**: Store the A/B/C classification in a native FUB custom field so it can be used for filtering/smart lists within FUB.
5.  **Audit Dashboard**: The Next.js app should have a "Live Feed" of AI decisions so agents can see the "brain" working in real-time.
